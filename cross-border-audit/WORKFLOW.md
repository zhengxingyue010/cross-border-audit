# 跨境电商交易审计工作流

## 项目概述
跨境电商交易订单的风险审计与刷单识别系统，带自动学习的风险模式库。

## 三个核心步骤

```
原始CSV → [步骤1: CSV预处理] → 清洗后数据 → [步骤2: 风险识别] → 风险标记数据 → [步骤3: 报告生成] → 最终审计报告
                          ↓                              ↓
                    清洗报告                      更新风险模式库（记忆）
```

---

## 详细步骤

### 步骤 1：CSV 预处理
**Skill:** `skills/csv-preprocessor/`

**输入：**
- 原始 CSV 文件（放在 `data/raw/` 目录）

**输出：**
- 清洗后 CSV：`data/processed/processed_文件名.csv`
- 清洗报告：`data/processed/清洗报告_文件名.md`

**执行内容：**
1. 读取并解析 CSV
2. 数据清洗（去重、处理缺失值）
3. 数据标准化（格式统一）
4. 数据验证

---

### 步骤 2：风险模式识别（核心 · 带记忆）
**Skill:** `skills/risk-detector/`

**输入：**
- 清洗后 CSV（来自步骤 1）

**输出：**
- 风险标记数据
- 新模式发现报告
- **更新的风险模式库**（如果有新发现）

**执行内容：**
1. **加载记忆** - 读取 `risk-patterns/patterns.json`
2. **已知模式匹配** - 应用所有已知风险模式
3. **新模式发现** - 统计分析、关联规则挖掘
4. **更新记忆** - 将新模式写入模式库（需确认）

**记忆机制：**
```
上次运行发现的新模式 → 自动成为这次的已知模式
```

---

### 步骤 3：风险评分与报告生成
**Skill:** `skills/report-generator/`

**输入：**
- 风险标记数据（来自步骤 2）
- 新模式发现（来自步骤 2）

**输出：**
- 最终审计报告：`reports/审计报告_YYYY-MM-DD_HHMMSS.md`

**执行内容：**
1. 风险汇总与评分
2. 生成详细报告
3. 提供改进建议

## 使用方式

### 完整流程运行
```
1. 把原始 CSV 放入 data/raw/
2. 说："运行跨境电商审计完整流程"
3. 等待生成报告
```

### 单独运行某一步
```
- "只做 CSV 预处理"
- "用现有数据做风险识别"
- "基于风险结果生成报告"
```

### 查看和更新模式库
```
- "显示当前风险模式库"
- "把这个新模式加入模式库"
- "查看历史经验教训"
```
